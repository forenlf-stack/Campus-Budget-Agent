# 省钱搭子：大学生消费决策 Agent

“省钱搭子”面向生活费相对固定的大学生，把消费管理从事后记账提前到消费前决策。系统长期保存每个用户自己的总预算、交易历史、消费偏好和餐食候选，在用户准备吃饭或购买零食饮料时给出具体建议，并且只有在用户最终确认后才写入支出。

## 当前实现

### 账户与数据隔离

- 支持邮箱注册、登录和退出。
- 密码使用带随机盐的 `scrypt` 哈希，数据库不保存明文密码。
- 登录使用随机会话令牌；浏览器 Cookie 设置为 `HttpOnly`、`SameSite=Lax`，数据库只保存令牌的 SHA-256 哈希。
- 每个用户拥有独立的资金设置、交易、退款、餐食候选、推荐决策和导入预览。
- 页面入口和所有数据 API 都会验证会话；数据查询和写入始终附带服务端确认的 `userId`，不接受客户端指定用户。

### 总预算制财务事实层

- 设置当前余额、月生活费、固定支出、储蓄目标、必要预留和本月总消费预算。
- 当前可用余额以手动保存值为基准，之后随收入/退款增加、随支出减少，并在每月生活费发放日自动增加。
- 分类只用于观察消费结构，不单独维护分类预算上限。
- 设置默认地点、正餐建议价格和参考上限、零食饮料频率/金额限制、口味与严格忌口。
- 手动新增、编辑收入、支出与退款；退款必须关联同一用户的原支出，删除会进入可恢复的回收站。
- 内置微信、支付宝、银行卡、现金账户，也可增加自定义账户；账户间转账独立保存，只改变账户余额，不计入收支和预算。
- 交易可绑定资金账户；历史记录仍允许保留为“未指定账户”，避免在没有证据时错误迁移。
- 自动计算实际净支出、总预算余额、每日可花金额、月底预计结余/超支趋势和分类消费占比。
- 支持最近账目一键复制，以及每周、每月、每年的固定支出/订阅账单和到期提醒。
- 支持文字、账单截图、Excel、CSV 和 TSV 的导入预览，用户校对后才批量写入；预览区可批量修改分类、商家、账户和固定支出属性。
- 导入时清洗美团等平台订单后缀，同时保留原始商家、项目和流水引用作为证据。
- 用户可选择记住“商家/项目 → 分类”的修改；后续导入自动套用，低置信度或重复记录仍集中进入复核区。

### 消费决策 Agent

- 从个人餐食候选库中按时段、地点、预算、口味、忌口和近期消费推荐餐食；默认返回最多 6 个、接口可请求 1～10 个候选。
- 支持菜单文字输入；配置多模态模型后可识别菜单图片，并要求用户确认不确定价格。
- 展示候选排序、风险、购买后预算影响和确定性 Skill 执行步骤。
- 支持围绕当前候选和个人财务背景继续与餐食 Agent 对话；保留最近 20 条消息，支持较完整的自然语言条件和多轮调整。
- 餐食确认后原子写入决策记录和 `AGENT` 来源支出，并使用幂等键防止重复记账。
- 零食饮料使用红/黄/绿分级判断；即使选择忽略提醒购买，也必须再次确认，确认后原子写入决策和交易。
- 账单分析展示最近 3、7、30、90 天趋势、主要分类和高支出日期/时段。

预算、金额、频率和日期计算全部由确定性 TypeScript 代码完成。LLM 只负责理解自然语言和组织解释；模型不可用时会退回本地规则，不影响核心预算计算和手工记账。

Agent 约束采用分层策略：被禁用的候选、明确过敏/绝对忌口和用户当次明确给出的价格上限会阻断；系统参考价、常用餐时段、默认地点、最近吃过和普通“不想吃”只参与降权与风险说明，不在检索阶段删除信息。所有消费仍必须由用户最终确认后写账，账户隔离、金额计算和幂等性不交给模型决定。模型请求具备有限重试和离线规则降级，菜单识别也保留更充足的处理时间。

## 技术栈

- Next.js 16 App Router、React 19、TypeScript
- Tailwind CSS 4
- SQLite、Prisma schema/migrations、`node:sqlite`
- Zod、Vitest

业务读写目前使用 `node:sqlite` 参数化 SQL；Prisma 负责 schema、迁移和 Client 生成。所有金额均以整数“分”保存。

## 环境要求与启动

- Node.js 22.5 或更高版本
- npm 10 或更高版本

```powershell
npm install
Copy-Item .env.example .env
npm run db:generate
npm run db:init
npm run db:seed
npm run dev
```

打开 `http://localhost:3000`。首次启动可以注册新用户，也可以使用种子演示账户：

```text
邮箱：demo@budget.local
密码：Demo1234
```

`npm run db:seed` 会重建本地演示数据和演示密码，请勿在需要保留本地账户或交易时执行。普通迁移使用 `npm run db:init`，不会清空数据。

## 模型配置

不配置模型也可以使用预算、交易、规则推荐、零食判断和菜单文字基础能力。模型设置页面可配置：

- DeepSeek 兼容接口：自然语言理解、对话和建议文案。
- GLM OCR：菜单文字识别。
- OpenAI 兼容多模态接口：菜单/账单图片理解。

API Key 仅写入项目根目录下被 Git 忽略的 `.local-model-config.json`。当前模型配置属于本地部署级配置，用户财务数据和决策数据仍按账户隔离。

## 当前边界

- 当前采用一个 SQLite 数据库，适合课程演示和单机部署；多实例部署前需将会话和导入预览迁移到共享存储。
- 登录为课程演示所需的本地邮箱密码方案，尚未实现邮箱验证、忘记密码、MFA、登录限流和管理员角色。
- 导入预览保存在当前服务进程内存中，30 分钟过期，且绑定创建它的用户。
- 日用品、游戏娱乐和充值订阅的专项消费决策尚未完成。
- 不读取支付宝、微信或银行账户，不自动支付、下单或转账，也不会绕过用户确认自动记账。
